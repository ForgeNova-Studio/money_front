# 앱 시작 안정화 티켓

## P0
### T1. 초기화 실패 UI 및 재시도 플로우
- 목표: 초기화 실패 시 복구 가능하게 만든다.
- 범위
  - Splash/초기화 화면에 재시도 버튼 추가
  - 네트워크 오류/치명 오류 메시지 구분
  - 재시도 시 초기화 Future 재실행
- 완료 기준
  - 실패 시 사용자 입력으로 재시도 가능
  - 재시도 성공 시 정상 라우팅

### T2. AppInit/Auth 상태 분리 및 라우팅 보장
- 목표: 앱 초기화 완료 후 인증 상태로만 화면 전환
- 범위
  - AppInit 상태와 Auth 상태를 명확히 분리
  - routerProvider에서 AppInit 완료 전에는 라우팅 지연
- 완료 기준
  - AppInit 미완료 상태에서 라우팅 변경 없음
  - AppInit 완료 후 Auth 기준으로만 홈/로그인 전환

## P1
### T3. 앱 시작 성능 측정 로깅
- 목표: 시작 성능 지표 확보
- 범위
  - 앱 실행 시각, 초기화 단계별 소요 시간 측정
  - 디버그/릴리즈 로그 레벨 분리
- 완료 기준
  - 시작~홈/로그인 진입까지 시간 로그 확인 가능

### T4. 크래시/리포팅 도입
- 목표: 초기화 오류 가시화
- 범위
  - Crashlytics 등 리포팅 도구 도입
  - 초기화 실패 예외 리포팅
- 완료 기준
  - 초기화 실패 이벤트가 대시보드에 기록됨

## P2
### T5. 스플래시 UX 보강
- 목표: 전환 안정성과 복귀 동작 정의
- 범위
  - 네이티브→Flutter 전환 깜빡임 최소화
  - 백그라운드 복귀 시 스플래시 표시 여부 정의 및 테스트
- 완료 기준
  - 복귀 시 상태 꼬임/백 화면 발생 없음

## 참고
- 우선순위는 배포 안정성 기준으로 산정
- 각 티켓은 별도 브랜치에서 개발 후 PR 제출
