홈 캐시 프리로드 티켓

배경
- 홈 진입 시 백엔드 응답 대기 동안 앱바 가계부 이름/월 데이터가 빈 상태로 보임
- 목표: 캐시된 가계부/월 데이터를 즉시 표시하고 백그라운드에서 최신 데이터로 갱신

T1. 캐시 우선 월 데이터 로딩
- 담당: FE
- 범위: Home ViewModel + Repository 캐시 조회
- 작업:
  - 진입/가계부 변경 시 캐시된 월 데이터를 먼저 읽어 상태에 반영
  - 캐시가 있으면 즉시 표시하고, 네트워크 갱신은 백그라운드로 수행
- 수용 기준:
  - 캐시가 있고 네트워크가 느려도 UI 공백 없이 데이터 표시
  - 캐시가 없으면 기존 로딩 동작 유지

T2. 로딩 UI 정책 변경
- 담당: FE
- 범위: Home 화면 위젯
- 작업:
  - 캐시가 있을 때 전체 로딩 스피너 숨김
  - 백그라운드 갱신 중에는 최소한의 인디케이터(앱바 스피너/토스트 등)만 표시
- 수용 기준:
  - 캐시가 있으면 앱바/캘린더가 빈 상태로 보이지 않음
  - 로딩 표시가 차단적이지 않음

T3. 캐시 유지 + 에러 처리
- 담당: FE
- 범위: Home ViewModel + UI 알림
- 작업:
  - 백그라운드 갱신 실패 시 캐시 데이터 유지
  - 에러는 스낵바 등 비차단 알림으로 표시
- 수용 기준:
  - 네트워크 실패 시에도 캐시 데이터가 유지됨
  - 에러가 사용자에게 전달됨

T4. 가계부 전환 UX
- 담당: FE
- 범위: 가계부 선택 + Home 갱신
- 작업:
  - 가계부 변경 시 해당 가계부 캐시 데이터가 있으면 즉시 표시
  - 이후 최신 데이터로 갱신
- 수용 기준:
  - 가계부 전환 시 캐시가 있으면 UI 공백이 없음

T5. 검증 시나리오
- 담당: QA/FE
- 범위: 수동 테스트
- 시나리오:
  - 캐시 있음 + 네트워크 지연
  - 캐시 있음 + 네트워크 실패
  - 캐시 없음 + 네트워크 정상
  - 가계부 전환(캐시 있음)
  - 가계부 전환(캐시 없음)
- 수용 기준:
  - T1~T4의 동작이 모두 충족됨
