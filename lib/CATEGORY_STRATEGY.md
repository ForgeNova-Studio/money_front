## 💡 카테고리 전략 아이디어

### 🅰️ 옵션 A: 기본 카테고리 + 커스텀 서브카테고리

```
📁 식비 (시스템 기본)
   ├── 점심 (사용자 추가)
   ├── 편의점 (사용자 추가)
   └── 배달 (사용자 추가)
```

**장점**:
- 통계 분석 시 일관성 유지 (메인 카테고리 기준)
- 사용자 커스터마이징 가능
- 기존 구조 변경 최소화

**단점**:
- 2단계 구조로 복잡도 증가

---

### 🅱️ 옵션 B: 완전한 사용자 정의 카테고리

```
사용자가 카테고리 전체를 직접 생성/수정/삭제
```

**장점**:
- 완전한 자유도
- 각자 라이프스타일에 맞춤

**단점**:
- 초기 설정 필요
- 사용자 간 통계 비교 어려움
- 빈 앱 상태로 시작 시 불편

---

### 🅲️ 옵션 C: 하이브리드 (추천 ⭐)

```
기본 카테고리 (삭제 불가) + 사용자 추가 카테고리
```

**예시**:
```
[기본] 식비, 교통, 쇼핑, 생활, 의료, 교육, 기타
[사용자 추가] 반려동물, 구독서비스, 경조사...
```

**장점**:
- 신규 사용자도 바로 사용 가능
- 필요 시 확장 가능
- 통계 기본 지원 + 커스텀

---

### 🅳️ 옵션 D: 태그 기반 시스템

```
지출: 배달음식 [#식비] [#야식] [#1인분]
```

**장점**:
- 하나의 지출에 여러 태그 할당 가능
- 다양한 관점에서 분석 가능

**단점**:
- 구현 복잡
- 사용자 입력 부담

---

### 🅴️ 옵션 E: AI 자동 분류 + 수동 수정

```
OCR 스캔 → AI가 카테고리 자동 추천 → 사용자 수정 가능
```

**장점**:
- 입력 편의성 극대화 (이미 OCR 기능 있음)
- 학습으로 정확도 향상

**단점**:
- AI 연동 필요
- 초기 정확도 이슈

## 📝 권장 사항

### 🅲️ 하이브리드 + 아이콘/색상 커스터마이징 (추천)

1. **기본 카테고리**: 9개 유지 (삭제 불가, 숨기기는 가능)
2. **사용자 카테고리**: 무제한 추가
3. **아이콘/색상**: 각 카테고리별 커스텀 가능
4. **순서 조정**: 드래그로 순서 변경

### 통합 카테고리 모델 제안

수입과 지출의 카테고리를 하나의 엔티티로 통합:

```dart
class Category {
  final String id;
  final String name;
  final IconData icon;
  final Color color;
  final CategoryType type; // income, expense
  final bool isDefault;    // 기본 카테고리 여부
  final int sortOrder;     // 정렬 순서
}
```
